# Prompt: Cybersecurity Learning Roadmap

**Use case:** Generate a structured, honest learning roadmap for transitioning into cybersecurity from a software/systems background. Avoids generic advice — forces the AI to account for what you already know and where you actually want to go.

---

## Prompt

```
I am a software engineering student with the following background:
- Strong C/C++ and systems programming experience (memory management, pointer arithmetic, low-level debugging)
- x86 Assembly knowledge
- AI-integrated web development (TypeScript, API integration)
- Currently teaching C programming in a Ministry of Defense cyber education program
- GPA: 85, 2 years into a B.Sc. Software Engineering degree

My target: [Choose one — Application Security / Penetration Testing / Reverse Engineering / Malware Analysis / Security Research / SOC/Blue Team]

My timeline: [e.g., 6 months / 1 year / until graduation]

My available time per week: [e.g., 5–10 hours outside of coursework]

---

Build me a learning roadmap with the following structure:

**Phase 1 – Foundation gaps** (what I likely don't know yet, given my background)
**Phase 2 – Core domain skills** (the specific technical skills required for my target)
**Phase 3 – Practical application** (labs, CTFs, projects that build a visible portfolio)
**Phase 4 – Certification / credential** (which cert makes sense at my stage, if any)

For each phase:
- List specific resources (not just "read books" — name the book, course, platform)
- Estimate time investment honestly
- Flag what I can skip because my systems background already covers it
- Identify the single most important thing to do in each phase

Don't give me a generic roadmap. Account for what I already know and what the market actually values right now for entry-level security roles.
```

---

**When to use this prompt:**
- At the start of a learning sprint to get a concrete plan rather than a vague list
- When re-evaluating your direction after 3–6 months
- Adapted for a friend or student you're mentoring into security
